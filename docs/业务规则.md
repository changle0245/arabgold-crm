# ArabGold CRM 业务规则

> 本文档记录系统中的关键业务约定和默认行为，供产品/业务/开发团队对齐。

---

## 客户阶段流转规则

客户阶段定义见 [`src/lib/constants.ts`](../src/lib/constants.ts) `STAGES` 数组：

```
待定 → 新接触 → 报价中 → 已寄样 → 已成交
                                    ↘
                                     沉默（任意阶段超 30 天无联系且未成交）
```

### 默认允许跳阶段

**业务员录入成交单 (`deal`) 时，客户当前阶段会自动推进到「已成交」，无论之前在哪个阶段。**

具体规则（见 [`supabase/migrations/20260517010000_record_stage_changes_in_triggers.sql`](../supabase/migrations/20260517010000_record_stage_changes_in_triggers.sql) `advance_stage_on_deal` 触发器）：

- 报价中 → 已成交  ✅ 允许（客户跳过寄样直接下单，常见于展会、老客户、小额订单）
- 已寄样 → 已成交  ✅ 允许（标准路径）
- 新接触 → 已成交  ✅ 允许（极端场景，但允许）
- 已成交 → 已成交  ✅ 触发器幂等

### 为什么不强制"先寄样后成交"

外贸场景里，以下情况都合法但跳过寄样：
- 展会现场下单
- 老客户复购（已知质量）
- 小批量试单（不需要样品）
- 客户提供自己的样品/规格

如果未来业务规则改变，需要强制"先寄样后成交"：
- **方案 A（弱提示）**：在 `src/components/deal-panel.tsx` 录入成交时检测 `customer.stage !== '已寄样'`，弹窗确认
- **方案 B（强制阻断）**：在 `advance_stage_on_deal` 触发器加 `if v_old_stage NOT IN ('已寄样') then raise exception ...`

## 自动提醒规则

提醒类型见 `src/lib/constants.ts` `REMINDER_TYPES`，10 种。其中**自动生成**的有：

| 类型 | 触发条件 | 默认到期 | 实现位置 |
|---|---|---|---|
| `follow_up` | 成交单 insert 后 | 成交日 + 30 天 | `auto_reminder_after_deal` |
| `payment` | 成交单 insert 且 `deposit_received=false` | 成交日 + 7 天 | `auto_reminder_after_deal` |
| `silent_customer` | 客户超 30 天无联系且未成交（手动/定时扫描） | 今天 | `scan_silent_customers()` |
| `reorder_cycle` | 已成交客户按采购频率到期 | 自动算 | `scan_reorder_cycle_reminders()` |

**去重规则**：以上自动提醒都按 `(customer_id, type, status, due_date, note)` 五元组去重，同条件不会重复插入。

业务员可手工建任意类型，无去重限制。

## 客户分级与价值

- `level` 字段：`L1`（高价值）/ `L2`（中）/ `L3`（低）/ `待定`
- 当前**没有自动评级逻辑**，全靠业务员/admin 手工标
- 未来若需 RFM 模型自动评级，应在 `customers` 表加 `level_auto` 字段，保留手工 `level` 作为覆盖

## 集中度风险

`get_concentration_risk_customers()` 函数返回近 12 个月营收占比超过 `system_settings.concentration_risk_threshold` 的客户。

- **默认阈值 10%**（已在 [`20260518040000_lower_concentration_threshold.sql`](../supabase/migrations/20260518040000_lower_concentration_threshold.sql) 由 30% 下调）
- 阈值由 admin 在老板大屏右上角"修改阈值"按钮调整，范围 5%-30%
- ⚠️ **多币种限制**：函数直接 `sum(deal_amount)`，不区分币种。当前 ArabGold 以 RMB 结算为主，跨币种成交少，可接受。若未来 USD/AED/EUR 成交比例上升，需在 `deals` 表加 `amount_cny`（按成交时汇率折算）或 RPC 函数加入汇率折算逻辑。

## 业绩目标

- `system_settings.monthly_revenue_target` 存月度目标（**单位：美元**）
- admin 在老板大屏右上"设置目标"按钮录入
- 进度条按 `monthDealAmount / monthlyTarget * 100%` 计算，超过 100% 变绿
- ⚠️ 同样不做币种折算

---

## 客户归属与转单

- 每个客户必有一个 `owner_id`（业务员）
- 业务员只能查看/编辑自己名下客户（RLS）
- admin 可在编辑客户页面修改 `owner_id`
- **转单审计**：每次 `owner_id` 变化，会自动 insert 一条 `customer_ownership_changes` 记录，详情页 timeline 可见

## 删除规则差异（不同实体）

| 实体 | 删除时是否阻塞 | 副作用 |
|---|---|---|
| 报价（quotation） | ⚠️ 如有子版本会被阻挡 | 必须先删新版本再删旧版本 |
| 成交（deal） | 无 | 客户累计成交额/次数自动重算（触发器 `refresh_customer_deal_stats`） |
| 样品（sample） | 无 | 无副作用 |
| 客户（customer） | 无 | 级联硬删 8 张子表（contact_logs / quotations / deals / samples / reminders / attachments / tags / stage_changes / ownership_changes）。删除按钮要求输入客户名作为二次确认。 |
| 附件 | 无 | 仅删 attachment 记录，Storage 文件未级联清理 |
