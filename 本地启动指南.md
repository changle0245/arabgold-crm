# ArabGold CRM · 本地启动指南（Docker 本地版）

> **方案**：本地 Docker 跑 Supabase 完整栈，零云端依赖。
> **首次耗时**：30-60 分钟（主要花在装 Docker + 拉镜像上）。
> **之后启动**：1-2 分钟。

---

## 第一步：安装 Docker Desktop（你来做，必须的）

1. 打开 https://www.docker.com/products/docker-desktop/
2. 点击 **Download for Windows - AMD64**（约 500MB）
3. 双击安装包运行
4. 弹出 "Configuration" 时：
   - ✅ 勾上 **Use WSL 2 instead of Hyper-V**（你的 WSL 已就绪）
   - 其他默认
5. 安装完成后**重启电脑**（强制）
6. 重启后从开始菜单启动 **Docker Desktop**
7. 弹出协议 → Accept
8. 等待主界面左下角出现 **绿色 "Engine running"** 字样
9. **保持 Docker Desktop 在后台运行**

---

## 第二步：首次启动（双击 start.bat）

1. 双击项目目录里的 `start.bat`
2. 脚本会自动：
   - 检查 Docker 是否运行 ✓
   - 安装 Node 依赖（约 1 分钟）
   - **首次启动 Supabase**：拉取约 3GB 的 Docker 镜像（5-15 分钟，看网速）
   - 自动把 Supabase 凭据写入 `.env.local`
   - 启动 Next.js dev server
   - 自动打开浏览器到 `http://localhost:3000`

3. 浏览器打开后会跳到登录页 —— **还没账号，先去创建管理员**

---

## 第三步：创建第一个管理员账号（双击 init-admin.bat）

**保持 start.bat 那个窗口别关**，再双击 `init-admin.bat`：

1. 输入管理员邮箱（任意，本地用，比如 `admin@arabgold.com`）
2. 输入密码（至少 6 位）
3. 输入姓名
4. 看到 `[√] 管理员账号创建成功` 即成功

回到浏览器登录页，用刚才的邮箱密码登录即可。

## 日常使用

| 操作 | 做法 |
|---|---|
| **启动系统** | 确保 Docker Desktop 在跑 → 双击 `start.bat` |
| **停止前端** | 关闭 start.bat 那个黑窗口，或按 Ctrl+C |
| **完全停止 Supabase**（释放内存） | 双击 `stop.bat` |
| **添加更多团队成员** | 系统内：左侧菜单 → 成员管理 → 新增成员（管理员账号才能看到） |
| **重置所有数据**（慎用） | 在项目目录开 PowerShell：`.\bin\supabase.exe db reset` |

## 资源占用提示

- Docker Desktop 后台运行：约 1-2 GB 内存
- Supabase 本地栈跑起来：约 1-2 GB 内存
- 暂时不用时双击 `stop.bat` 可以释放

## 常见问题

**Q: Docker Desktop 启动失败 / WSL 报错？**
A: 打开 PowerShell（管理员）运行 `wsl --update`，再重启 Docker Desktop。

**Q: `supabase start` 卡在拉镜像？**
A: 国内网络可能慢。让它跑就行，期间可以喝杯水。

**Q: 端口 3000/54321/54323 被占用？**
A: 找出占用进程关掉，或在 `supabase\config.toml` 改端口。

**Q: 关电脑前要不要先停 Supabase？**
A: 不强求，但建议运行一下 `stop.bat`，下次启动更干净。
